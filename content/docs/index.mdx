In this session, we'll make an AI agent durable using the Workflow SDK. We're starting from a working agent that can chat, search the web, run commands in a cloud sandbox, and maintain persistent memory — and we'll upgrade it so it can run for hours, survive request timeouts, and resume seamlessly.

Let's start by setting up our project.

<Steps>
<Step>
Deploy this template.

<a href="https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fpranaygp%2Fworkflow-agent-starter" target="_blank" rel="noopener noreferrer">
  <img src="https://vercel.com/button" alt="Deploy with Vercel" />
</a>
</Step>
<Step>
Clone the new project's repository locally.

```bash
git clone https://github.com/your-username/workflow-agent-starter.git
cd workflow-agent-starter
```

</Step>
<Step>
Install the Vercel CLI globally.

```bash
npm i -g vercel
```

</Step>
<Step>
Link to your Vercel project.

```bash
vercel link
```

</Step>
<Step>
Pull environment variables from Vercel.

```bash
vercel env pull
```

</Step>
<Step>
Install dependencies.

```bash
pnpm install
```

</Step>
<Step>
Start the development server.

```bash
pnpm dev
```

</Step>
</Steps>
